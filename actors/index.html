Movies and Actors

The actors and movies dataset was very large and made network visualisation in d3 time difficult. There were too many nodes for d3.js to handle so I reduced the dataset to be based around a small group of actors that I knew acted in movies together.

I began by making some visualisations in Tableau to answer simple questions about the dataset.

Tableau Visualisations

Proportional Representations

This tree map attempts to answer the question: "Which of these movies had the greatest number of actors in the dataset?" Since the data remains in the range of 2 - 7 I feel that the tree map effectively encodes the data. It fits a lot of information into a small space. Using a bar chart for this information was less effective since it results in too many bars.

D3.js Visualisations

The most interesting feature of this dataset is the level of interconnectedness between actors. In order to show the relationships between actors I decided to use two types of visualisation; a network graph and an adjacency matrix.

I decided to use these representations as they are both obviously good candidates for representing relationships in a network. Both of these visualisations required me to write scripts to parse the data into lists of nodes and links. While I had to write a lot of code to get the network visualisation to work. The adjacency matrix visualisation code was taken directly from here.

Here's the Adjacency Matrix visualisation.

I developed 3 network visualisations. The first is the simplest and only involves movies and actors in the set given above. The second shows all the additional movies an actor acted in. The third network visualisation includes additional actors that are in the movies from visualisation one. This has a lot more nodes and is a better example of visual art than something that conveys a lot of information.

One "Easter Egg" I have added to these visualisations is that when a reader clicks on a node it will bring them to the Wikipedia page associated with that node. This allows the reader to easily drill down further into the information presented by the visualisation. For a variety of reasons it doesn't work in every case but it works in most cases.
